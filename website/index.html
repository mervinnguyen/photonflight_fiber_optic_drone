SYSTEM OVERVIEW

01
🚁
QUADCOPTER PLATFORM
YoungRC 8" X-frame with Matek F405-WMN flight controller, Hobbywing XRotor G2 Micro 65A 6S 4-in-1 ESC, and four LigPower F100 (2810) FPV motors with Gemfan 8145 3-blade props. Powered by Zeee 6S 5200mAh 100C LiPo.
02
🧠
ARDUPILOT FLIGHT CONTROL
Matek F405-WMN running ArduCopter v4.5.7 firmware. BLHeli_32/AM32 ESC firmware on HobbyWing XRotors. MAVLink 2.0 over UDP/TCP for telemetry and command, bridged by Raspberry Pi 4B via UART.
03
💡
FIBER-OPTIC TETHER
Bidirectional 1000BASE-BX SFP BiDi transceivers (TX 1310nm / RX 1550nm) over a single-strand OS2 simplex LC-UPC SMF fiber patch cable — 150ft flown in demonstration. Full-duplex gigabit, zero EMI, unjammable.
04
👁️
RASPBERRY PI AI CAMERA
IMX500 sensor with onboard AI accelerator running YOLOv2 at up to 35 FPS — no external compute. PiCamera2 + TensorFlow Lite runtime. JSON output → pymavlink for autonomous yaw tracking. 75.86% object detection accuracy over 6 trials.
05
📡
VIDEO & TELEMETRY PIPELINE
GStreamer 1.22+ H.264 encoding pipeline at 720p/30fps, transported over RTP/UDP through fiber to Mission Planner v1.3. MAVLink telemetry channel runs simultaneously on the same fiber via TCP/IP static IP. 2–4ms latency.
06
🛡️
3D PRINTED ASSEMBLIES
Custom SolidWorks-designed, FDM-printed parts on Prusa i3 MK3S+ via PrusaSlicer: fiber spool mechanism (underside-mounted), media rack housing Raspberry Pi + SFP converter, battery mount with GPS/compass standoff, ESC and camera brackets.

  • H.264 video encoded by GStreamer 1.22+ on RPi
  • RTP/UDP stream → GStreamer on Windows (Mission Planner)
  • Video + telemetry run simultaneously on same fiber
  • RTP/RTSP and H.264 encoding per IETF standards
2–4ms latency | 720p/30fps
MAIN POWER RAIL
  • Zeee 6S 5200mAh 100C LiPo — 22.2V nominal
  • XT90 connectors for main power distribution
  • Battery → Flight Controller → 4-in-1 ESC
  • ESC BEC supplies 5V to Matek F405 FC
  • ~20 minute flight time demonstrated
  • ISDT BattGo BG-8S battery checker / monitor
DC-DC CONVERSION
  • Mini MP1584EN DC-DC Buck converter (×2)
  • Steps down LiPo voltage to power SFP media converters
  • Raspberry Pi powered from ESC 5V BEC rail
  • Twisted-pair signal wires per IEC 61000 EMI rules
  • Soldered joints (high-current) + crimped (signal)
  • IPC-610 soldering quality standards applied
MAVLINK v2 FRAME FORMAT
  • STX | LEN | INC/CMP FLAGS | SEQ | SYS ID | COMP ID | MSG ID
  • PAYLOAD (0–255 bytes) | CHECKSUM (2 bytes) | SIGNATURE (13 bytes)
  • UDP and UART are different protocols — same MAVLink frame bytes
  • RPi "repackages" (not translates) data between UART and UDP
  • Works like CAN bus — agreed format, sender/receiver encode/decode
  • MAVLink 2.0 frame: 12–280 bytes total
pymavlink | pyserial
MISSION PLANNER CONNECTION
  • Mission Planner v1.3 connects via UDP on open port
  • Static IP assigned to Raspberry Pi on ground network
  • RPi opens UDP port → listens for Mission Planner commands
  • Bidirectional: Commands from MP → FC via RPi → UART
  • Telemetry from FC → UART → RPi → UDP → Mission Planner
  • GStreamer Runtime on Windows renders H.264 inside MP HUD
Mission Planner 1.3 | ArduPilot
VIDEO ENCODING STACK
  • libcamera / rpicam-apps v1.11.1 captures frames from IMX500
  • GStreamer 1.22+ pipeline: capture → H.264 encode → RTP mux
  • FFmpeg for additional transcoding/container handling
  • UDP stream output over Gigabit Ethernet link
  • Fiber SFP converts Ethernet to OS2 optical signal
  • Ground GStreamer Runtime decodes and renders in Mission Planner
H.264 | RTP/RTSP | UDP
NETWORKING STANDARDS
  • IEEE 802.3 Gigabit Ethernet (1000BASE-X)

  • SFP MSA (Multi-Source Agreement) transceiver compatibility
  • RTP/RTSP and H.264 encoding per IETF/ISO standards

// COMMS DESIGN RATIONALE

FIBER-OPTIC ADVANTAGE

💡 PHOTON FLIGHT — FIBER-OPTIC (1000BASE-BX WDM)
✅Zero EMI susceptibility — single-mode OS2 fiber immune to all RF jamming and electronic warfare
✅Full-duplex 1 Gbps — H.264 720p video + MAVLink telemetry + AI data simultaneously on one fiber
✅2–4ms end-to-end video latency demonstrated in flight over 150ft tether
✅WDM BiDi transceivers: TX 1310nm / RX 1550nm — full duplex on a single strand (no second cable)
✅No RF emissions — invisible to passive signal detection and direction-finding equipment
✅Galvanic isolation — eliminates ground loops and noise near high-current ESCs/motors
✅Operates in GPS-denied and damaged infrastructure environments
📻 TRADITIONAL RF RADIO LINK (2.4/5.8 GHz)
❌Susceptible to jamming — 2.4 GHz and 5.8 GHz bands are easily blocked by adversaries
❌Bandwidth-limited — video and telemetry compete; heavy compression degrades quality
❌Higher latency and jitter — multipath reflections, interference degrade real-time control
❌RF emissions expose operator position to direction-finding equipment
❌Signal quality degrades with distance and obstacles — unreliable in hazardous environments
❌Signal can be intercepted, replayed, or spoofed by adversaries
❌GPS dependence — fails in signal-denied environments (rubble, jamming, underground)
